# NPC Portrait Renderer (server)

Server-side stack that renders NPC portraits for the Android app. Runs as a Docker
Swarm stack on the Hetzner box (`ritalee`), fronted by caddy-docker-proxy at
**`https://npc-fast.colman.com.br`** (HTTP basic auth, user `npc`).

## Architecture

```
Android app ──HTTPS+basicauth──> caddy ──> queue (FastAPI FIFO) ──> fastsd (FastSD CPU)
                                            /submit  /status/{id}      /api/generate
```

- **`queue`** (`queue_app.py`, `Dockerfile.queue`) — a small FastAPI FIFO proxy.
  FastSD renders one image at a time, so the queue accepts submissions immediately,
  assigns a position, and drains them serially. The app shows "you're #N in line"
  and polls for the result. Public (via caddy); this is the only exposed service.
  - `POST /submit` → `{ job_id, ahead }`
  - `GET /status/{job_id}` → `{ state, ahead, queue_length, image, error }`
    (`state` = `queued | processing | done | error`; `image` is base64 PNG when done)
- **`fastsd`** (`Dockerfile`) — [FastSD CPU](https://github.com/rupeshs/fastsdcpu),
  internal only. The queue forwards each request body **verbatim** to its
  `POST /api/generate`, so the **app chooses the model per request** — the server
  keeps no fixed model.

## Model / licensing

The app has ads, so the render config must be commercially licensed. The app sends
**DreamShaper-8** (`Lykon/dreamshaper-8`, CreativeML OpenRAIL-M) + **LCM-LoRA**
(`latent-consistency/lcm-lora-sdv1-5`, openrail++), 4 steps, tiny autoencoder — see
`PortraitQueueClient` in the app. Because the queue forwards the body untouched,
switching models is an app-side change only; no redeploy here.

> Note: `deploy.sh`'s warm-up/smoke-test still calls the older SDXS-512 OpenVINO
> model, which is **non-commercial** — fine for a local timing check, but do not use
> it for anything user-facing.

## Deploy

```bash
# On ritalee, from this directory:
bash deploy.sh            # build images + render compose + deploy + warm up + timed test
bash deploy.sh rebuild    # rebuild + redeploy only
```

`deploy.sh` renders `docker-compose.yml` (which has a `__BCRYPT__` placeholder) into
`docker-compose.rendered.yml` by substituting the bcrypt hash from `auth.txt`
(doubling `$` so `docker stack deploy` doesn't interpolate it), then deploys the
`rpg-npc-fastsd` stack.

The HF model cache is bind-mounted at `hf/` so models survive restarts; the first
render after a fresh model pull is slow (download + pipeline build), then ~30s warm.

## Secrets (git-ignored)

- **`auth.txt`** — basic-auth `user` / `password` / `bcrypt`. Generated by `deploy.sh`
  on first run if absent (or copy `auth.txt.example`). The app must be built with the
  same password via `-PnpcImagePassword=...`.
- **`docker-compose.rendered.yml`** — contains the real bcrypt hash; regenerated by
  `deploy.sh` every deploy.

Never commit either. See `.gitignore`.
